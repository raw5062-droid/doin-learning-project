# AGENTS.md

## Working style
- 모든 설명은 한국어로 한다.
- 나는 코딩 초보다.
- 코드를 쓰기 전에 먼저 3단계 이하의 짧은 계획을 제시한다.
- 한 번에 크게 바꾸지 말고 작은 단위로 나눈다.
- 새 라이브러리나 패키지를 추가하기 전에는 이유를 설명하고 승인을 요청한다.
- 파일 수를 최소화한다.
- 변경 후에는 실행 방법과 테스트 방법을 아주 쉽게 설명한다.
- Windows 기준으로 설명한다.

## Project rules
- 첫 버전은 가능한 한 단순하게 만든다.
- 로그인, 결제, 배포, 서버 구축은 내가 따로 요청하기 전에는 하지 않는다.
- 치명적 버그, 데이터 손실 가능성, 불필요한 복잡도를 우선적으로 줄인다.

## Review guidelines
- 깨지는 기능, 저장 실패, 삭제 오류, 콘솔 에러를 우선적으로 본다.
- 초보자가 실행하기 어려운 구조라면 문제로 표시한다.
- README에 실행 방법이 없으면 문제로 표시한다.
